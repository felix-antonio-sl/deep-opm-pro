Patrones y antipatrones

Moldes de modelado que se repiten y errores canónicos que la validación diagnostica; solo las invariantes declaradas bloquean.

La forma

Estructura de un patrón

Un patrón publicable declara: intención · cuándo usar · cuándo NO usar · OPD mínimo · OPL esperado · reglas propietarias · brechas · ejemplo.

Cita sus reglas por URN, no las copia. Una plantilla de selección reutiliza contenido local del modelo; una Pieza es una entidad externa gobernada y puede anclarse a una biblioteca (ver hoja «Anclar y calcar»).

Catálogo (§9)

Patrones candidatos

  • Sistemas sociotécnicos · agente/rol/autoridad
  • Decisión con supervisión humana · interfaz crítica
  • Control loop · estados ortogonales
  • Composición por interfaz (subsistemas)
  • Configuración y tradeoffs · digital twin

Anti-patrón · categoría

Proceso vs objeto

  • AP-12 estados de proceso — los estados son de objetos → descompón en subprocesos o usa atributo.
  • AP-26 objeto transiente creado-y-consumido sin observación → usa invocación.
  • AP-25 proceso de soporte/mantenimiento sin esfuerzo sostenido → enlace estructural.

Anti-patrón · agencia

Agente ≠ instrumento

  • AP-05 agente conectado a software / robot / IA / máquina → usa enlace de instrumento.

Agente = persona o grupo de personas; una organización solo califica cuando el nombre denota explícitamente ese colectivo humano. Las abstracciones organizacionales y los sistemas de información son objetos, no agentes.

Anti-patrón · refinamiento

Refinar mal

  • AP-13 refinamiento de un solo hijo → ≥2, o placeholder tipificado.
  • AP-16 refinamiento cíclico → rompe el ciclo.
  • AP-06 consumo/resultado en el contorno de un proceso descompuesto → al primer / último subproceso.
  • Agregación de cajón universal → usa el enlace estructural correcto.

Anti-patrón · nombres y estados

Nombrar mal

  • AP-22 sinónimos múltiples para la misma cosa → nombre canónico (usa la ontología).
  • AP-14 duplicar estados para simular inicial+final → marca el único como ambos.
  • Estado sin atributo que lo dimensione (A5).

Anti-patrón · control

Control mal puesto

  • AP-01/02 c/e sobre un resultado → muévelo al lado de entrada.
  • AP-09/10 c/e sobre estructural o invocación → error de categoría.
  • AP-04 resultado directo al estado inicial → al rectángulo o a un estado no-inicial.

Detalle en la hoja «Control de flujo».

La cura de fondo

Heurísticas (A5–A8)

  • Modela primero el cambio de valor.
  • Nombres que digan qué transforma o qué es.
  • Registra supuestos y brechas; no plasmes barro.
  • A8 separa bloqueadores estructurales · mejoras metodológicas · estilo/legibilidad.
  • Quien conoce el dominio confirma, aparte, si el modelo representa y sirve.
