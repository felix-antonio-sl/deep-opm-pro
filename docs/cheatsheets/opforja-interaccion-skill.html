Hablar con la skill

Cómo darle buen input a la skill de modelado modelamiento-opm, y por qué cuando te bloquea está haciendo su trabajo.

Su carácter

Dialéctica, no complaciente

La skill bloquea ante barro y exige aclarar antes de plasmar. No es fricción: es que no construye sobre supuesto. Cuando te para con una pregunta, respóndela — no la rodees.

Quién hace qué

El reparto

Tú traes el dominio y el gusto; la skill trae el método. La autoridad sobre el significado del dominio sigue siendo humana: la skill no inventa cuáles son los procesos, agentes o valores correctos.

Buen input

Dale un propósito claro

Una oración verbo-objeto del cambio, no de la forma («mejorar disponibilidad de camas», no «crear una plataforma»). Responde sus preguntas de aclaración una a una.

La procedencia decide

Cuándo manda el proto

Si el modelo tiene sello de procedencia porque nació de un proto OPL-ES, el proto es la fuente y la mesa es su proyección: las correcciones se re-elicitan y recompilan. Si el modelo nació en la mesa (table-origin), el push libre es legítimo. En ambos carriles, la autoridad del dominio sigue siendo humana.

Declara tu régimen

Dile en qué estás

  • «Esto es un apunte» → acompaña sin bloquear (relaja el cierre).
  • «Exploro bottom-up» → no fuerza el SD-primero (relaja el arranque).

Diálogo asíncrono

Notas y ratificación

  • Notas de mesa — preguntas ancladas a un componente; la skill las consume al re-elicitar.
  • [RATIFICAR] norma — exige una fuente verificable antes de pasar a vigente.
  • [RATIFICAR] hipótesis — exige evidencia o decisión explícita, más alcance declarado.

El transporte

El puente

Con acceso al host, la skill usa el puente directo:

cd app && bun run mesa pull <modelo>
cd app && bun run mesa push <modelo> bundle.json --nota "…"

Repite el pull inmediatamente antes del push, revisa el delta con una persona y no confíes en el 409 para detectar un bundle nacido de un pull antiguo.

Sin él, usa el portapapeles: Ctrl+K › «Copiar contexto para la skill» (W6.0). Mismo bucle, transporte manual.

Expectativas honestas

No pidas lo imposible

La mesa no hace: out-zoom automático, diff de modelos, PDF real, cosimulación federada, multiusuario, ni inferencia automática. Modela y valida; no infiere ni razona por ti.
