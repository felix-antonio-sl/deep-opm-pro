## Brechas declaradas, no calendarizadas

- Láminas didácticas OPD para el manual OPM puro: generarlas desde un proto canónico mediante el renderer headless, con fuente reproducible. No dibujarlas manualmente ni mezclar este corte pedagógico con correcciones doctrinales.
- Auth/tenants v2: invitaciones, roles efectivos y administración multiusuario; abrir solo ante demanda nombrada.
- Out-zoom: capacidad ausente y declarada en `registro-conformidad-ssot.md`; no hay modelo productivo que la exija.
- Simulación probabilística sin pesos: requiere variante de modelo, serialización, interfaz, OPL y diagnóstico; esperar caso real.
- Anclaje: verbo formal `promover-a-Pieza`, resolución externa completa, herencia mutacional y OPL/render del metacontrato; mantener como evolución, no como deuda urgente.
- GAPs visuales de `spec-forja-opd-es` §22: abrir por evidencia de usuario o incumplimiento de gate, no por inventario.
- Auth/identidad v1, mobile de solo lectura, puente mesa↔skill, Taller, vitrina, modo apunte, gesto de anclar, C4, Inspector, chrome de gestión y frontera `src/app` están cerrados; no reaparecen como backlog.

## Decisiones de convergencia que no abren épica

- No ejecutar el refactor radical propuesto en sesión. Las únicas dependencias inversas reales ya se corrigieron y L7–L8 las sellan; un nuevo movimiento exige dolor medido o una feature bloqueada.
- No implementar una vía adicional de graduación desde `PanelCarpetas`: el callback cuestionado no forma parte de la ruta productiva actual.
- No convertir todas las observaciones estéticas de sesión en backlog. Solo abre corte una brecha con evidencia de usuario, decisión de producto o gate reproducible.

## Gates por corte

- Cambio documental: enlaces locales activos sin roturas, punteros raíz exactos, fechas ISO y una sola versión visible por especie.
- Cambio de código: `cd app && bun run check` como mínimo.
- Cambio visual: añadir `bun run lint`, `bun run build`, `bun run design:governance` y los E2E afectados.
- Refactor transversal o cierre de corte: `bun run gate:refactor`, con el servidor de desarrollo apagado antes del smoke.
- Cambio doctrinal: flujo del custodio en `kora-pneuma`, re-pin de puentes y `bun run cordon:skill`.

## Criterios de selección

1. Una brecha abre corte solo con usuario, decisión o gate afectado.
2. Cada corte declara beneficiario, decisión soportada y prueba que puede fallar.
3. El resultado se registra en un handoff nuevo; el roadmap vigente no se convierte en bitácora.
4. Un sucesor de este documento se publica como `roadmap-AAAA-MM-DD.md` y desplaza esta versión a `_archivo/`.
